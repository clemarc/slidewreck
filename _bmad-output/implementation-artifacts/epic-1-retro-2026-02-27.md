## Successes

1. **AD-5 Validation (Anthropic-only with built-in tools)** — Eliminated Story 1.3 entirely. Researcher agent handles existing talk discovery and stat extraction natively through built-in tool orchestration. Simpler architecture, less code.

2. **Native tooling understanding reduced complexity** — Deeper platform understanding led to simpler solutions, not more abstraction layers. This was the project lead's top takeaway.

3. **Reusable gate factory design** — `createReviewGateStep` in Story 1.6 is genuinely reusable for Gate 2 (Phase 2) and Gate 4 (Phase 5) without modification.

4. **Steady test growth with TDD discipline** — Test suite grew 7 → 29 → 50 → 86 → 99 across 5 stories. Never a broken build. TDD red-green-refactor consistently applied.

5. **100% story completion, zero critical issues** — All code review findings were MEDIUM severity.

---

## Challenges

1. **Mastra API documentation lag (3/5 stories)** — Every story introducing a new Mastra API required verifying from installed type definitions. Examples: `PostgresStore` `id` parameter, sub-path imports (`@mastra/core/agent`), `mastra.getWorkflow()` vs nonexistent `getWorkflows()`.

2. **Defensive validation not encoded in specs** — Input validation hardening appeared as code review feedback in 3/5 stories. Missing `.positive()`, empty string rejection, fetch timeouts — all preventable if specs included validation requirements.

3. **No integration tests** — All 99 tests are unit/structural. Pipeline never tested end-to-end with mocked LLM responses. Manual Studio testing was the only E2E verification.

4. **Uncommitted changes on main** — `researcher.ts` and `web-search.ts` have uncommitted modifications.

5. **Zod v4 peer dependency warnings** — `@ai-sdk` packages expect zod v3. Non-blocking but unresolved.

---

## Key Insights

1. **Platform understanding leads to simpler architecture** — Leaning on native tooling eliminates custom code. AD-5 proved this conclusively.

2. **API verification from type definitions is mandatory** — Mastra docs lag behind installed versions. Must verify import paths, constructor params, and return types from `.d.ts` files before implementing.

3. **Specs that encode defensive patterns eliminate review round-trips** — Every MEDIUM code review finding in Epic 1 would have been prevented by a validation checklist in the story spec.
