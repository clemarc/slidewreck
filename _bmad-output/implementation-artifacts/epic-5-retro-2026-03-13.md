## Challenges

1. **Context payload size for Designer agent not anticipated by spike** — The design didn't account for the accumulated workflow state filling the Designer agent's context window. Required reworking the flow to trim what gets passed to the agent. Spike should have tested with realistic payload sizes.

2. **Anthropic API structured output limitations** — The Zod schema used in-app couldn't be passed directly to the LLM. Schema adaptations were needed for Anthropic API compatibility (e.g., simplified schema for structured output vs full validation schema). This is a recurring pattern that needs documentation.

3. **ESM-only mermaid-cli** — Required dynamic `import()` instead of static import. Workaround is functional but inconsistent with rest of codebase.

4. **Puppeteer version mismatch** — mermaid-cli expects `^23`, puppeteer v24 installed. Works at runtime but fragile on dependency updates.

5. **Chromium-dependent tests skipped in CI** — renderMermaid integration tests use `describe.skipIf` for environments without Chromium. Coverage gap for CI.

---

## Key Insights

1. **Spikes need to test actual LLM calls with realistic payloads** — API shape verification is necessary but not sufficient. The spike should estimate context payload size and test structured output schema compatibility with the actual Anthropic API, not just Zod validation.

2. **Structured output schema for LLM may differ from in-app Zod schema** — Anthropic API has limitations on what schema shapes it accepts for structured output. Document the "dual schema" pattern: full Zod schema for app validation, simplified schema for LLM calls.

3. **Pattern reuse is the velocity multiplier** — Epic 5's single-day delivery was possible because Epics 1-4 established clean foundations. Investment in good patterns pays compound returns.

4. **Low-priority action items need deliberate cleanup** — Monitoring items accumulate across retros. Barry quick-flow pass is the mechanism to triage: resolve, drop, or justify continued carry.

5. **One-epic-per-sprint cadence continues to prove its value** — Spike → sprint planning → stories → retro → gap for action items. The rhythm is working.
