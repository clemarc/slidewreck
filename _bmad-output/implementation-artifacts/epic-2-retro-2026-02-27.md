## Successes

1. **100% Epic 1 action item follow-through** — All 6 items completed. Defensive Validation Checklist eliminated entire categories of review findings. Integration tests (Story 2-0) became the foundation for the entire epic.

2. **Composite step pattern discovery** — Stories 2-3 and 2-4 both adapted the architecture's suggested APIs (`.dountil()`, `.branch()`) to a simpler composite step with re-suspend. Well-reasoned, documented, and reused across stories.

3. **Integration test infrastructure** — `InMemoryStore` from `@mastra/core/storage` eliminated Docker dependency for tests. Mock `createStep()` pattern enabled true integration testing without agent internals. Both patterns reused in every subsequent story.

4. **Steady test growth with TDD discipline** — 106 → 113 → 139 → 147 → 153 → 157 across 5 stories. Never a broken build.

5. **Clean stories when following established patterns** — Stories 2-1 and 2-2 were almost mechanical, following patterns from Epic 1. Zero review findings.

---

## Challenges

1. **Architecture-vs-reality gaps (2/5 stories)** — Architecture prescribed `.dountil()` for loopback and `.branch()` for conditional logic. Both had unclear or overly complex semantics when combined with `suspend()`/`resume()`. Dev had to invent composite step pattern during implementation rather than during planning.

2. **Persistent researcher model tier reversion (3/5 stories)** — `researcher.ts` OPUS→SONNET fix kept reverting during squash merge rebases. Appeared in Stories 2-0, 2-1, 2-2 before direct fix on main (`57413b8`). Root cause: fixing main-level bugs on feature branches that get squash-merged.

3. **Agent mocking exploration (Story 2-0)** — Three attempts before finding the right pattern. `vi.spyOn(researcher, 'generate')` failed (internal `stream()` call), mock model replacement timed out, mock `createStep()` finally worked. This was an unplanned spike within a story.

---

## Key Insights

1. **Encoding review finding patterns into checklists shifts quality left** — Epic 1's Defensive Validation Checklist eliminated validation findings in Epic 2. The same approach should be applied to suspend/resume edge cases.

2. **Verify APIs during planning, not during dev** — Architecture-vs-reality gaps in Stories 2-3 and 2-4 would have been caught by a minimal verification spike during sprint planning.

3. **Composite step pattern is the real architecture for suspend/resume flows** — Simpler than `.dountil()` and `.branch()` when combined with `suspend()`/`resume()`. Architecture doc should be updated to reflect this.

4. **100% action item follow-through is achievable and compounds** — Each retro's improvements make the next epic cleaner.
