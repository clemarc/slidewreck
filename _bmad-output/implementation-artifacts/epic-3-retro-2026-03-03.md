## Successes

1. **100% completion with zero review findings** — Defensive Validation Checklist from Epic 1 continues to pay dividends. No stories required rework after review.

2. **RAG infrastructure pattern reuse** — `MDocument → chunk → embedMany → pgVector.upsert()` established in Story 3-1, cleanly reused in 3-2 and 3-3. Later stories were almost mechanical.

3. **43 new tests with TDD discipline** — 157 → 173 → 191 → 200 across 3 stories. Never a broken build. Mock patterns (PgVector, embedMany, fs/promises, fetch) carried forward from Epic 2.

4. **Graceful degradation and backward compatibility** — Per-material failure capture in `failed` array (NFR-8). All new schema fields use `.optional().describe()`. No breaking changes.

5. **Error handling production-quality** — URL timeout via AbortController (30s), individual material failures isolated, pipeline continues without throwing.

---

## Challenges

1. **Unplanned OpenAI dependency** — Anthropic has no embedding API. Had to introduce `@ai-sdk/openai` with `text-embedding-3-small` (1536 dimensions). Not caught during planning because no API verification spike was performed.

2. **PDF support dropped against PRD** — `MDocument` from `@mastra/rag` has no `fromPDF()` method. FR-5 explicitly requires PDF support. Shipped without it — gap against the PRD.

3. **Model tier reversion (3rd consecutive epic)** — `researcher.ts` OPUS→SONNET kept reverting during rebases. Flagged in all 3 story dev notes. Root cause: git flow rule not executed, no automated test guard.

4. **Delete+recreate as upsert workaround** — `pgVector.upsert()` creates duplicates with same metadata. Both knowledge bases use drop+recreate pattern. Works for small datasets but won't scale.

5. **Circular dependency in workflow** — Cannot import pgVector from `index.ts` in workflow due to circular imports. PgVector instantiated inline as workaround — multiple instantiation points.

6. **Mastra Agent class limitation** — `agent.instructions` not publicly accessible. Tests had to read source files directly to verify instruction content.

---

## Key Insights

1. **API verification spikes during planning would have caught both the embedding provider gap and PDF gap** — Two architectural surprises that were fully avoidable with 30 minutes of investigation.

2. **Deferring action items across back-to-back epics has compounding costs** — Model tier reversion persisted 3 epics because the git flow fix was never executed. Process fix needed, not willpower.

3. **One-epic-per-sprint with retro+actions gap prevents this structurally** — Cadence: Epic done → Retro → Execute action items → Sprint planning → Next epic. No skipping steps.

4. **Per-run isolation accidentally enables embedding provider swaps** — The delete+recreate pattern means switching embedding models doesn't require migration. Different dimensions just work on next run.

5. **Pattern reuse from early stories accelerates later stories** — Investment in clean infrastructure in Story 3-1 paid compound returns in 3-2 and 3-3.

## Significant Discoveries

**Embedding Provider Gap:** Architecture assumed Anthropic-native stack. Anthropic has no embedding API, forcing OpenAI dependency. Research needed into local/alternative providers before this becomes entrenched.

**PDF Support Gap:** FR-5 requires PDF support. MDocument API lacks `fromPDF()`. Pre-processing approach identified (external parser → `fromText()`) but not yet implemented.

**Impact on Epic 4:** Neither discovery changes Epic 4's direction (evals don't depend on embedding provider or PDF). However, the API verification spike for Mastra evals is critical — Epic 4 is built entirely on an unverified API surface.
